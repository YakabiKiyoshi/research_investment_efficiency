# 学術論文の引用内容妥当性検証をバックグラウンドで実行

1 本または複数の `.tex` ファイルに対し、本文中で言及される全引用文献を、対応する PDF・日本語訳 (.jp.tex / .jp.pdf) を実際に読み込んだ上で、**本文の主張と引用先文献の内容が整合しているか・著者の論点を正しく要約しているか・ページ参照は正しいか・適用範囲を超えた拡張解釈をしていないか・文献の主張と相反する内容を引用していないか**を徹底検証し、結果を `doc/citation_check_X.txt` に保存する。

`proofread-paper` / `math-proof-check` との役割分担：

- `proofread-paper`: 文章表現・表記揺れ・引用の体裁（著者名表記・年・誌名略称・ページ番号の書式）
- `math-proof-check`: 証明の論理的厳密性・命題と証明の整合・数値計算の独立検証
- **`citation-check`（本スキル）**: 引用先文献を実際に読み込んだ上での内容的妥当性（主張の事実性・方向性・適用範囲・過剰解釈）

実行は **バックグラウンドの Agent** に委譲する。スキル本体（メイン会話）はファイル番号の決定・プロジェクト設定の読込み・Agent 起動のみ行なう。引用キー数が多い場合は文献群を分割して**複数 Agent を並列起動**する。

## 引数

- `$ARGUMENTS`（省略可）: 検証対象の `.tex` ファイル（スペース区切りで複数指定可、相対または絶対パス）。
- 省略時は **`<cwd>/.claude/citation_check.md`**（または `<cwd>/.claude/proofread.md`）から対象ファイルおよび文献探索ディレクトリを取得。
- いずれも存在しない場合はユーザーに対象ファイルを確認。

## 絶対の制約

- **対象の .tex ファイルは絶対に直接編集しない**。検証結果は `doc/citation_check_X.txt` に保存するのみ。
- LaTeX のビルド・コミット・プッシュもしない。
- バックグラウンドで実行する（メイン会話をブロックしない）。
- 修正・ビルド・コミット・プッシュは、レポート確認後にユーザーが明示的に指示した場合のみ別ステップとして実行。
- 文献 PDF が手元にない場合は「未取得」と明記し、内容照合は留保（推測・捏造しない）。
- 時間がかかってもよいので、各引用箇所と文献本文の対応を徹底することを優先する。

## 手順

### Step 1. プロジェクト設定の読込み

1. `$ARGUMENTS` に対象ファイルの明示があればそれを使う。
2. なければ `<cwd>/.claude/citation_check.md`、なければ `<cwd>/.claude/proofread.md` を Read する。
3. いずれも存在しない場合はユーザーに対象ファイルを確認。

### Step 2. 出力ファイル番号 X の決定

- `<cwd>/doc/citation_check_<n>.txt`（n は自然数）を Glob で列挙
- 未使用の **最小の** 自然数 X を決定
- `doc/` ディレクトリが存在しない場合は Agent 側で作成

### Step 3. 引用キーの抽出

対象 `.tex` から `\cite`, `\citep`, `\citet`, `\citetalias`, `\citepalias`, `\citeauthor`, `\citealp`, `\citealt`, `\nocite` の全形式（optional argument `[...]` を含む）の引用キーを抽出し、重複削除した一覧を作成：

```bash
grep -hoE '\\(cite|citep|citet|citetalias|citepalias|citeauthor|citealp|citealt|nocite)[a-z]*\*?(\[[^]]*\])?\{[^}]+\}' *.tex \
  | grep -oE '\{[^}]+\}' | tr -d '{}' | tr ',' '\n' \
  | sed 's/^[[:space:]]*//;s/[[:space:]]*$//' | sort -u
```

### Step 4. 文献ファイルの所在特定

各キーについて以下を順に探す（プロジェクト設定で順序を上書き可能）：

1. `<cwd>/references/{key}.jp.tex`（日本語訳があれば最優先；高速かつ正確）
2. `<cwd>/references/{key}.jp.pdf`
3. `<cwd>/references/{key}.pdf`（原文 PDF）
4. `<cwd>/references/documents/{key}.pdf`（政府レポート等）
5. `<cwd>/../papers/{first_letter}/{key}.pdf`（papers がアルファベット別フォルダ構造の場合）
6. `<cwd>/../papers/{first_letter}/{key}.jp.tex`
7. `<cwd>/../papers/{first_letter}/{key}.jp.pdf`
8. `<cwd>/doc/`, `<cwd>/old/` も補助的に探索

ジャーナル略称のゆらぎ（例: `_RJE` ↔ `_RAND`, `_TAR` ↔ `_AcctRev`）も緩めに照合。完全一致が見つからない場合は近傍探索（著者・年が一致する候補を提示）。

### Step 5. 引用箇所の網羅収集

各キーについて、`grep -n {key} *.tex` で行番号付きの全引用箇所を取得し、前後 ±3 文の文脈を抽出する。

### Step 6. 引用キー数による分割

引用キーが 15 件を超える場合は、複数 Agent に分担させる：

- 12 件以下 → 単一 Agent
- 13〜25 件 → 2 Agent
- 26〜45 件 → 3 Agent
- 46 件以上 → 4 Agent

分担は引用キーの**アルファベット順**ではなく**論文内で重要な文献を均等に分配**する（先行研究の中心的文献を 1 Agent に集中させない）。各 Agent には独自の出力ファイル（`doc/citation_check_X_part1.txt` 等）に保存させ、すべての Agent 完了後にスキル本体が `doc/citation_check_X.txt` に統合する。または各 Agent が seek-and-append でロックなし排他的に同一ファイルに追記する。本スキルでは単一の `doc/citation_check_X.txt` への統合方式を推奨。

### Step 7. バックグラウンド Agent の起動

`Agent` ツールを以下のパラメータで複数並列に呼び出す：

- `subagent_type: general-purpose`
- `description: "Citation check round X part k"`
- `run_in_background: true`
- `prompt`: 下記「Agent への指示テンプレート」に、以下を埋め込んで渡す：
  - `X`（ラウンド番号）
  - 担当する引用キー一覧
  - 各キーの対応する .tex 引用箇所（行番号・文脈付き）
  - 各キーの文献ファイル所在（絶対パス、見つからない場合は「未取得」）
  - 対象 `.tex` ファイルの絶対パス一覧
  - 出力先（部分ファイルの絶対パス、または共有ファイルへの append 指示）
  - プロジェクト固有設定の内容

### Step 8. ユーザーへの通知

Agent 起動後、即座に報告：

- バックグラウンドで第 X 回引用チェックを開始したこと
- 引用キーの総数・取得済み件数・未取得件数
- 起動した Agent 数と担当分割
- 出力先（絶対パス）
- 完了時に通知される旨

Agent の進捗を polling しない。

### Step 9. 統合（複数 Agent の場合）

全 Agent 完了後、`doc/citation_check_X_part*.txt` を `doc/citation_check_X.txt` に統合する：

- ヘッダ（ラウンド・日付・対象・総数）は 1 つにまとめる
- 各 Part の本体（個別キーごとの照合結果）を順に結合
- 末尾にスキル本体が「総合サマリー」を追記

- ~/.claude/CLAUDE.md（ユーザー共通ルール）
- <cwd>/CLAUDE.md（プロジェクト固有ルール）
- <cwd>/.claude/citation_check.md または <cwd>/.claude/proofread.md
- 過去ラウンドのレポート（<cwd>/doc/citation_check_*.txt）— 過去の指摘との重複を避けつつ、独立視点で再検証
- ユーザー auto memory（~/.claude/projects/<project-slug>/memory/ 配下）

【検証観点】

## A. 主張の事実性
- 「X は Y を示した」「X は Y を主張する」と引用される内容が、文献の本文中に実際に存在するか
- 結論部・要旨・本文の主張と一致しているか
- 文献内で言及はされているが筆者の主張ではない内容（先行研究紹介・反対論等）を、文献筆者自身の主張として引用していないか

## B. ページ参照の正確性
- `\citep[p.~XX]{...}` のページ番号で示される主張が、当該ページに実在するか
- 範囲指定 `[pp.~5--7]` の妥当性

## C. 方向性の正誤
- 「増加」「減少」「正の関係」「負の関係」「上昇」「低下」「促進」「抑制」などの方向性が文献の結論と一致するか
- 主効果と副次効果の取り違え

## D. 適用範囲のずれ
- 文献の対象（業界・期間・サンプル・地域・契約タイプ等）と引用文脈で示唆される範囲の整合
- 「全産業」「全期間」のような一般化が文献のサンプル範囲を超えていないか

## E. 過剰一般化・拡張解釈
- 文献が条件付き・限定的に示した結論を、無条件・普遍的な結論として引用していないか
- 文献の数値・統計を、論文のモデル・パラメータと別の意味で使用していないか

## F. 対立内容の混入
- 文献の主張と本文の文脈が論理的に対立していないか
- 「整合する」「裏づける」と書きながら実は文献が反対方向の結論である事例

## G. 引用の必要性・適合性
- 引用が文脈で意味を持つか（読者が文献を参照することで本文の主張が補強されるか）
- より適切な文献が存在しないか（誤った文献を典拠としていないか）

## H. 文献メタデータの整合
- 著者名・年・誌名が bib エントリと一致するか（proofread-paper の領域だが、明らかな不整合は記録）
- 第一著者の表記揺れ（Heese / Heese, A. 等）

## I. 自己引用・arXiv 版との混同
- 同一論文の WP 版・査読付き版を混同していないか
- 著者の他論文との取り違え

## J. 文献内容そのものの問題（参考的指摘）
- 文献の主張に対する論文内の評価が客観的か（不当な美化・矮小化はないか）— ただし内容の妥当性自体は論文の責務ではなく、引用の使い方の問題に限って指摘

## K. 数値の引用整合（政府レポート・統計）
- レポートに記載される数値（金額・件数・割合）を本文の対応箇所と独立に照合
- レポートの集計対象・通貨単位・実質名目区分の取り違え

【重大度分類】
- **Critical**: 事実誤認・主張の方向性の正反対・存在しない主張への帰属（読者を誤導する重大な引用）
- **Major**: 適用範囲のずれ・過剰一般化・ページ番号の誤り・対立内容の混入
- **Minor**: 著者名・年の表記揺れ・軽微な数値ずれ・引用の必要性疑義
- **Style**: 引用の体裁の改善余地（proofread-paper の領域だが気づいたら記録）

【手順】
1. 担当する引用キー一覧を確認
2. 各キーについて、文献の中核（要旨・序論末尾の貢献宣言・結論・主要命題のステートメント）を Read で読む
3. 本文の引用箇所（行番号付き、前後 ±3 文）を Read し、引用文脈を把握
4. 引用文脈と文献内容を A〜K の観点で照合
5. 問題があれば重大度分類して記録
6. 問題なしであっても「✓」マークと根拠（引用先の該当箇所）を簡潔に記録
7. 文献が未取得の場合は「未取得」と記し、引用箇所のみ列挙

【独立視点】（過去ラウンドとの差別化）
過去ラウンドが既に存在する場合は、別の切り口で独立検証すること：
- ラウンド N-1 が「主張の事実性中心」ならラウンド N は「ページ番号・数値の独立再検証中心」
- ラウンド N-1 が「全文献俯瞰」ならラウンド N は「中心的文献の深堀り」

## `.claude/citation_check.md` の共通形式

プロジェクト固有情報は `<cwd>/.claude/citation_check.md` に書く（`<cwd>/.claude/proofread.md` を流用可）：

```markdown
# プロジェクト引用チェック設定

## 対象ファイル（相対パス）
- defense.jp.tex
- online_appendix.jp.tex

## 文献探索ディレクトリ（優先順）
- references/
- references/documents/
- ../papers/  # アルファベット別 A-Z フォルダ

## キーのジャーナル略称ゆらぎ（参考）
- _RJE ↔ _RAND（Rand Journal of Economics）
- _TAR ↔ _AcctRev（The Accounting Review）

## 既知の未取得文献（PDF 取得不能・書籍等）
- Demski_Feltham_1976_book（書籍、紙媒体で所蔵）

## 引用上の重点監視項目
- DCAA / DoD / GAO レポートの統計数値の照合
- Heese and Pérez-Cavazos (2019) の実証範囲（連邦全機関 296 件、防衛特化と誤認しない）
- Laffont and Tirole (1986) の最適メカニズム結果との照合
```

スキル側はこのファイルを無加工で Agent プロンプトに埋め込む。

## 関連スキル

- `proofread-paper`: 文章表現・引用の体裁の校正。引用先文献の内容には踏み込まない。
- `math-proof-check`: 数学的厳密性。引用文献の内容との照合は範囲外。
- **組み合わせ運用推奨**: 大改訂後は `math-proof-check` → `proofread-paper` → `citation-check` の順で 3 層検証することで、論理・体裁・引用妥当性をそれぞれ独立に保証できる。
