# 数学記号のリネーム・呼称統一・宣言追加

LaTeX 論文（とくに証明を含む数学的・経済学的論文）における**ノーテーション衛生（notation hygiene）**作業を、影響範囲の調査から再タイプセット検証まで段取りで進めるスキル。

## 起動条件

ユーザーが次のいずれかを依頼したとき：

- 「$X$ を $Y$ に変えてください」「$X$ と $Y$ を統一してください」
- 「同じ意味の記号が別名で呼ばれています、整理してください」
- 「$\mathrm{xx}$ の意味を注記してください」
- 「添字の使い分け（行動ラベル vs 偏微分など）を明確にしてください」
- 「装飾（$\hat{}$, $\bar{}$, $\check{}$, ...）の使い回しが混乱を生んでいるので整理してください」
- math_proof_check / proof_reading レポートの記号関連 Minor/Style 指摘を反映するとき

## 絶対の制約

- **数学的内容を変更しない**: 命題の主張・証明の論理は維持。本スキルは表現の整理に限定
- **同じ装飾を異なる意味に使い回さない**: $\hat{}$ が閾値、$\bar{}$ が上限、のように装飾の意味論を一貫させる
- **同記号を別の意味で再利用するときは、明示的な流用宣言を入れる**: 「ほかの変数と同様、両者を同じ記号で表わす」のような断り書きを節導入で明示
- **機械置換の副作用に注意**: `\partial\check S` → `\partial S^\dagger` で `\partialS` のようなコマンド名衝突が発生する例あり。置換後はかならずビルド確認
- **ビルド・コミット・プッシュは別ステップ**: スキル単独で完結させず、最後にユーザー確認を経て `commit-push` 等に引き継ぐ

## 想定する 4 つの典型ケース

### A. シンボルのリネーム（$X \to Y$）

例: $\bar S \to S^\star$、cite key `Mirrlees_1999_RES → Mirrlees_1975_Mimeo`

### B. 呼称の統一（同じ量を 2 通りに呼んでいる）

例: 「$\mu$ の第1表現／第2表現」と「$G$ の H 側項／L 側項」が同じ式形を指している

### C. 添字・略号の意味注記

例: `\mathrm{ch}` の意味（characteristic curve）を初出箇所に注記

### D. 添字記法の宣言（複数系統が併存）

例: $M_q$（偏微分）と $M_H$（行動ラベル）が同じ字 $M$ に乗っているとき、初出時に「以下、添字は偏微分」と宣言

## 手順

### Step 1. 影響範囲の調査

- `grep -nE` で対象記号・呼称の全出現を網羅。display 数式と inline、本文と証明、figure 注記まで含める
- 同じ装飾を持つ他の記号（衝突候補）を `grep` で確認
- 提案された新記号がすでに別の意味で使われていないか確認
- プロジェクトの記法 memory（`feedback_notation.md` などがあれば）と照合
- bib エントリの cite key を変える場合は、`.tex` 全体に対して `grep -c` で参照数を数える

### Step 2. 再利用 / 衝突回避の判断

選択肢を列挙してユーザーに合意を取る:

- **完全リネーム**: 新規記号に置き換え。装飾規約と整合させる
- **同記号の意図的再利用**: 異文脈で同じ役割を果たすときのみ。明示的な流用宣言を必須
- **新装飾の追加**: 装飾の意味論が一貫するか確認（記法 memory を更新）
- **注記のみ**: 略号添字や添字系統の併存は、初出箇所の宣言文だけで対応可能な場合が多い

機械的展開（例: $M_q$ を $\partial M/\partial q$ に全置換）の見積もりも提示する。100 件以上 + ヤコビアン折り返しが頻発する場合は、宣言文 1 つで予防するほうが軽量。

### Step 3. 置換の実行

- 機械的に安全な箇所は `Edit` の `replace_all`
- 文脈依存の修正（説明文の調整、`\equiv` → `=` の使い分け、注記の冗長性除去など）は個別 `Edit`
- インライン `$...$` 内の `\frac` 禁止 / `/` 推奨ルール（CLAUDE.md）に注意
- bib cite key を変える場合は `.tex` 側の `\citet`/`\citep` も同時更新（`replace_all`）

### Step 4. 注記・宣言の追加

ケース別の典型文例:

**(A) 同記号の異文脈再利用の流用宣言**（節導入や定義の直後）:
> これは無条件版の系\ref{cor:perfect}で完全シグナルのもとでのベンチマークとして定義した $S^\star$ と同じ役割を果たす。本研究では、ほかの変数と同様に、両者を同じ記号 $S^\star$ で表わす。

**(B) 略号添字の意味注記**（初出箇所のかっこ注記）:
> $\gamma_L = \gamma_L^{\mathrm{ch}}(\gamma_H)$ を一意に定めることを示す（添字 $\mathrm{ch}$ は $G=0$ の特性曲線（characteristic curve）を意味する）。

**(C) 添字記法の宣言**（最初の偏微分が登場する直前）:
> 以下、誘因両立制約のスラック関数 $M$ と均衡トレードオフ条件 $G$ の偏微分を添字で表わす（$M_q \equiv \partial M/\partial q$、$G_{\gamma_H} \equiv \partial G/\partial\gamma_H$ など）。$M_H$、$M_L$ は行動 $\alpha_H$、$\alpha_L$ のもとでのマネージャー期待利得であり、$M$ の偏微分ではない。

**(D) 呼称の統一定義**（初出時の命名節）:
> 以下、$\mu$ の第 1 の表現（$\gamma_H$ のみに依存し、$G$ の第 1 項に等しい）を H 側項、第 2 の表現（$\gamma_L$ のみに依存し、$G$ の第 2 項に等しい）を L 側項と呼ぶ。

### Step 5. タイプセット検証

- `make` または `lualatex <メインファイル>.tex` 等でビルド
- log の Warning（未定義参照、引用切れ、`\partialS` のような未定義コマンドの兆候）をチェック
- `grep` で旧記号・旧呼称の残存ゼロを確認（とくに `replace_all` で取り切れない記号後の修飾は要注意）
- ページ数・PDF サイズの大幅変動がないか確認

### Step 6. 後追いチェック・引き継ぎ

- 大きな改変は `math-proof-check` で数学整合性を再確認
- 文章表現が伴う改変は `proofread-paper` で校正
- 装飾規約や呼称の memory を更新（`feedback_notation.md` 等）
- 反映が固まったら `commit-push`

## チェックリスト

リネーム・統一・宣言の事前確認:

- [ ] 全出現を `grep` で列挙したか（数式環境内・本文中・bib 含む）
- [ ] 提案された新記号が別の意味で使われていないか
- [ ] 装飾規約・記法 memory と矛盾しないか
- [ ] 同記号の異文脈再利用なら、流用宣言の場所を決めたか
- [ ] 機械置換の副作用（コマンド名衝突など）を確認したか
- [ ] bib cite key を変えるなら、.tex 側 `\citet`/`\citep` も同時更新するか

事後確認:

- [ ] `grep` で旧記号・旧呼称の残存ゼロ
- [ ] `make` ビルド成功
- [ ] log に Warning なし
- [ ] 引用・参照整合（`\ref`, `\cite`）
- [ ] 装飾規約 memory を更新したか

## 関連スキル

- `math-proof-check` — 改変後の数学的厳密性検証（Critical/Major/Minor/Style 検出）
- `proofread-paper` — 文章表現・引用・体裁の校正
- `commit-push` — 反映後のコミット・プッシュ

## ノーテーション衛生で頻出する観点（参考）

過去のセッションで実際に発生した、見落としがちな事項:

- **装飾の使い回し禁止**: $\hat\gamma_H$ を $\bar\gamma_H$ の代替として提案するのは NG（$\hat{}$ は閾値専用）
- **同記号の異文脈再利用**: 経済的解釈が一致するときに限り許容。$\bar S$（条件付き版）を $S^\star$（無条件版と同記号）に統一した例
- **添字系統の併存**: $M_H$（行動ラベル）と $M_q$（偏微分）が同じ字に乗る場合は宣言で予防
- **略号添字の意味**: `\mathrm{ch}`（characteristic curve）のような略号は形式（立体化）と意味（注記）を別個に扱う
- **冗長な formula 反復**: 呼称統一後は「$G$ の H 側項 $(...)$」のような formula を毎回書く必要はない。命名定義で省略可能
- **`\equiv` 規約**: CLAUDE.md ルールでは「`\equiv` は定義を表わす」。同じ記号を 2 回 `\equiv` するのは規約上は望ましくないので、後発側を `=` に変える、あるいは流用宣言で正当化する
- **bib エントリのプレプリント・再録**: mimeo + reprint 形式は `@unpublished` + note に reprint 情報、あるいは `@article` + note に circulation 情報の 2 形式。原典性を強調するなら前者
