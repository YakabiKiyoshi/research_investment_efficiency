# 論文の徹底校正をバックグラウンドで実行

1本または複数の `.tex` ファイルに対し、**文章表現・引用文献・引用統計・数学的内容・数値計算結果・LaTeX形式**を徹底的にチェックし、結果を `doc/proof_reading_X.txt` に保存する。

実行は**バックグラウンドの Agent** に委譲する。スキル本体（メイン会話）はファイル番号の決定・プロジェクト設定の読込み・Agent 起動のみを行なう。

## 引数

- `$ARGUMENTS`（省略可）: 校正対象の `.tex` ファイル（スペース区切りで複数指定可、相対または絶対パス）。
- 省略時は **`<cwd>/.claude/proofread.md`** をプロジェクト設定として読み込み、そこに列挙された対象ファイルを使用する。
- `.claude/proofread.md` も存在しない場合はユーザーに対象ファイルを確認する。

## 絶対の制約

- **対象の .tex ファイルは絶対に直接編集しない**。校正結果は `doc/proof_reading_X.txt` に保存するのみ。
- LaTeX のビルド・コミット・プッシュもしない。
- バックグラウンドで実行する（メイン会話をブロックしない）。
- 修正・ビルド・コミット・プッシュは、校正レポート確認後にユーザーが明示的に指示した場合のみ別ステップとして実行する。
- 時間がかかってもよいので徹底することを優先する。

## 手順

### Step 1. プロジェクト設定の読込み

1. `$ARGUMENTS` に対象ファイルの明示があればそれを使う。
2. なければ `<cwd>/.claude/proofread.md` を Read する。
   - 存在すれば、そこに記載された対象ファイル・プロジェクト固有の既知数値・追加ルールを取得。
   - 存在しなければユーザーに対象ファイルを確認。

### Step 2. 出力ファイル番号 X の決定

- `<cwd>/doc/proof_reading_<n>.txt`（n は自然数）を Glob で列挙
- 未使用の**最小の**自然数 X を決定
- `doc/` ディレクトリが存在しない場合は Agent 側で作成する

### Step 3. 付属ファイルの自動検出

対象 `.tex` と同じディレクトリを調査：

- **`.bib` ファイル**が存在すれば、引用文献メタデータ・引用統計のチェックを有効化
- 対象ファイルが複数あり、いずれかの .bib を共有するならそれを bib チェック対象にする

### Step 4. バックグラウンド Agent の起動

`Agent` ツールを以下のパラメータで呼び出す：

- `subagent_type: general-purpose`
- `description: "Thorough paper proofreading round X"`
- `run_in_background: true`
- `prompt`: 下記「Agent への指示テンプレート」に、以下を埋め込んで渡す：
  - `X`（ラウンド番号）
  - 対象 `.tex` ファイルの絶対パス一覧
  - `.bib` ファイルの絶対パス（検出できた場合）
  - `<cwd>/.claude/proofread.md` の内容（プロジェクト固有の既知数値・追加ルール・優先指摘事項）
  - 出力先の絶対パス `<cwd>/doc/proof_reading_X.txt`

### Step 5. ユーザーへの通知

Agent 起動後、即座に報告：

- バックグラウンドで第 X 回校正を開始したこと
- 出力先（絶対パス）
- 完了時に通知される旨

Agent の進捗を polling しない。

【対象】
{{target_tex_files_abs_paths}}

【.bib ファイル】（存在する場合のみチェック）
{{bib_file_abs_path_or_none}}

【出力先】
{{output_file_abs_path}}

【プロジェクト固有設定】
{{contents_of_cwd_dot_claude_proofread_md_or_none}}

【絶対の制約】
- 対象の .tex ファイルは絶対に直接編集しない。Edit/Write は出力ファイル（proof_reading_X.txt）にのみ使用する。
- LaTeX のビルド・コミット・プッシュはしない。
- 時間がかかってもよいので徹底することを優先する。

【参照すべき情報】
- ~/.claude/CLAUDE.md の「Writing Style and Terminology」「Mathematical notation」「Statistical terminology」「Accounting terminology」「Number style」などのユーザー共通ルール
- <cwd>/CLAUDE.md のプロジェクト固有ルール
- <cwd>/.claude/proofread.md のプロジェクト設定（既知数値・追加ルール・優先事項）
- <cwd>/doc/proof_reading_*.txt の直近ファイル（過去の指摘と重複を避けるため）
- ユーザー auto memory（~/.claude/projects/<project-slug>/memory/ 配下）

【校正観点】

## A. 文章表現（日本語）
CLAUDE.md の表記ルールに従い、以下を Grep で網羅検出：
- 表記ゆれ：「行う」→「行なう」／「表す」→「表わす」／「従う」→「したがう」／「特に」→「とくに」／「単に」→「たんに」／「仮に」→「かりに」／「米国」→「アメリカ」
- 学術用語：「ROE」→「ROCE」／「裁量的発生高」→「裁量的アクルーアル」／「利益調整」→「利益マネジメント」／「サンプル数」→「サンプル・サイズ」または「観測数」
- カタカナ複合語の中黒（「キャッシュ・フロー」など）。固有名詞・学術用語として定着したものは例外として明示
- 序数・回数・次数：「第一に」→「第1に」／「一次」→「1次」／「二回微分」→「2回微分」
- 文体の一貫性（である調の混在）、文法的誤り、不自然な表現、typo、全角/半角の混在

## B. 数学的内容
Grep で以下の違反を網羅検出：
- `\ge` / `\le` / `\geq` / `\leq` → `\geqslant` / `\leqslant` へ
- インライン数式 `$...$` 内の `\frac` → `/` へ（ディスプレイ数式内 `\frac` は OK、手動判定）
- `e^` の指数表記 → `\exp(...)`
- `\ln` → `\log`
- `\E[`, `\Pr[` の角括弧 → 丸括弧 `(...)`
- `\text{s.t.}` → `\text{subject to}`
- 参加制約タグ `IR` → `PC`
- 略語的な複数文字記号（IC, FFP, HN, LN, HD, LD, PC, CE 等）が `\mathrm{}` で立体になっているか
- 命題・補題・系・定義の番号整合性（本文と付録の `\ref{}` 相互参照）
- `\label{}` の重複、未定義参照 `\ref{}` / `\eqref{}`
- 証明の論理的整合性、未定義記号、ステップの飛躍
- 仮定（asm1, asm2 等）の独立性・必要性が証明内で参照されているか
- KKT条件・ラグランジアンの符号・比較静学の符号・極限挙動
- プロジェクト設定の「ノーテーション整理方針」に従う書き換え残り

## C. 数値計算結果
- プロジェクト設定の「既知数値」リストがあれば、各ファイル内の対応箇所と照合
- 感応度分析表・ケース比較表の数値間の整合性（行・列の合計、差分等）
- 必要に応じて Python/Julia で再計算して検証（一時ファイルは /tmp/ に置く）
- 小数点・桁数の整合性

## D. 引用文献（.bib が検出された場合のみ）
- `\citet{}` / `\citep{}` の使い分け（著者名が主語なら `\citet`、括弧内なら `\citep`）
- `\citealias{}` / `\citeasalias{}` 等のタイポ
- 各 bib エントリの著者・年・誌名・巻・号・頁・出版社・月の正確性と統一性
- 疑わしい場合は WebSearch で原典確認
- Title Case、ピリオド、address/publisher の分離

## E. 引用統計（.bib が検出された場合のみ）
- 本文中の `\cite` 系コマンドと .bib エントリの対応表
- 未引用エントリの検出
- 同一著者・年の重複や誤参照

## F. LaTeX 形式
- `\begin{}` / `\end{}` の対応
- `siunitx` の S 列記述、アスタリスクは `table-space-text-post={$^{***}$}` と `0.070{$^{***}$}` 形式
- 表注は `\multicolumn` ではなく tabular 外の minipage
- `align` 環境（`equation` ではなく）
- クロスファイル参照（付録側での本文ラベルへの参照等）は単独コンパイル時に未解決となる可能性を指摘
- **Overfull \hbox（テキスト幅超過）**: 対象 `.tex` と同名の `.log` ファイル（直近のビルド産物）を Read で取得し、`Overfull \hbox (Xpt too wide)` を全件抽出。X（オーバー幅）の降順で並べ、以下のカテゴリと優先度で分類：
  - **`detected at line N`** ≒ ディスプレイ数式（`align`/`equation` 環境）の幅超過
  - **`in paragraph at lines M--N`** ≒ 散文中のインライン数式・長い英単語・図（`\includegraphics`）等の幅超過
  - 優先度：≥ 20pt = 高、5–20pt = 中、< 5pt = 低（散文 hyphenation 由来の微小オーバーは情報提供のみ）
  - 各オーバーフローについて、対応行を Read で確認し、新記号を導入しない修正案を 1〜2 案提示する：
    - 行分割：`\notag\\` ＋ `&` ＋ `\quad`/`\qquad` インデント
    - 括弧サイズ：`\left[...\right]` を `\bigl[...\bigr]` に
    - 微小調整：`\!`（負の thin space）を演算子・括弧に挿入
    - 代数的同値変形（既存記号のみ。新変数導入は不可）
    - インライン数式が長すぎる場合：`align*` 等の display math 化
    - 図：`\includegraphics[width=\linewidth]{...}` で幅制限
  - `.log` が存在しない場合は、本スキルでは検出できない旨と「ユーザー側で一度 LaTeX をビルドしてから再実行」を案内する（**スキル本体・Agent 共にビルドしない**）。

【手順】
1. 対象 .tex をすべて全文 Read（長大なら offset/limit で複数回に分割）

8. 数値例は Bash で Python/Julia 実行して検証
9. レポートを Write で出力ファイルに保存

## `.claude/proofread.md` の推奨フォーマット（プロジェクト側で用意）

プロジェクト固有情報は `<cwd>/.claude/proofread.md` に以下の形式で書く：

```markdown
# プロジェクト校正設定

## 対象ファイル（相対パス）
- title_page.jp.tex
- defense.jp.tex
- online_appendix.jp.tex

## 既知数値（各ファイル内の一貫性チェック用）
- θ* ≈ 0.69（表値は 0.6885）
- h_F=0.45 での V=4.1795, α_L*=0.738, ...

## ノーテーション方針（最新の整理）
- 情報状態添字：\Omega（旧 \omega）
- 目標原価：\tau_\Omega（旧 T_\omega）
- 旧表記の書き残しは優先度高で指摘

## 追加の優先指摘事項
...
```

スキル側はこのファイルを無加工で Agent プロンプトに埋め込む。
