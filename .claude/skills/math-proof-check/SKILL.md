# 数学論文の数学的厳密性検証をバックグラウンドで実行

1 本または複数の `.tex` ファイルに対し、**証明の論理・命題と証明結論の整合・仮定の必要性・陰関数定理/KKT 条件/ラグランジアンの正しさ・境界ケース・数値カリブレーションの独立再計算・参照整合性**を徹底的に検証し、結果を `doc/math_proof_check_X.txt` に保存する。

`proofread-paper` と役割分担：
- `proofread-paper`: 文章表現・表記揺れ・引用文献メタデータ・LaTeX 体裁の校正
- `math-proof-check`（本スキル）: 証明の論理的厳密性・数学的整合性・数値計算の独立検証

実行は**バックグラウンドの Agent** に委譲する。スキル本体（メイン会話）はファイル番号の決定・プロジェクト設定の読込み・Agent 起動のみを行なう。

## 引数

- `$ARGUMENTS`（省略可）: 検証対象の `.tex` ファイル（スペース区切りで複数指定可、相対または絶対パス）。
- 省略時は **`<cwd>/.claude/proofread.md`** をプロジェクト設定として読み込み、そこに列挙された対象ファイルを使用する（proofread-paper と共通）。
- `.claude/proofread.md` も存在しない場合はユーザーに対象ファイルを確認する。

## 絶対の制約

- **対象の .tex ファイルは絶対に直接編集しない**。検証結果は `doc/math_proof_check_X.txt` に保存するのみ。
- LaTeX のビルド・コミット・プッシュもしない。
- バックグラウンドで実行する（メイン会話をブロックしない）。
- 修正・ビルド・コミット・プッシュは、レポート確認後にユーザーが明示的に指示した場合のみ別ステップとして実行する。
- 時間がかかってもよいので徹底することを優先する。

## 手順

### Step 1. プロジェクト設定の読込み

1. `$ARGUMENTS` に対象ファイルの明示があればそれを使う。
2. なければ `<cwd>/.claude/proofread.md` を Read する。
   - 存在すれば、そこに記載された対象ファイル・プロジェクト固有の既知数値・追加ルールを取得。
   - 存在しなければユーザーに対象ファイルを確認。

### Step 2. 出力ファイル番号 X の決定

- `<cwd>/doc/math_proof_check_<n>.txt`（n は自然数）を Glob で列挙
- 未使用の**最小の**自然数 X を決定
- `doc/` ディレクトリが存在しない場合は Agent 側で作成する

### Step 3. 付属ファイルの自動検出

対象 `.tex` と同じディレクトリおよびプロジェクト直下を調査：

- **`.bib` ファイル**があれば引用統計・被引用ページ指定の整合チェックに利用
- **数値検証スクリプト**（`bin/verify_*.jl`, `bin/*_calibration.py` 等）があれば独立再計算のクロス・チェック材料として利用
- 対象ファイルが複数あり、いずれかの .bib / 検証スクリプトを共有するならそれを検査対象にする

### Step 4. バックグラウンド Agent の起動

`Agent` ツールを以下のパラメータで呼び出す：

- `subagent_type: general-purpose`
- `description: "Thorough math proof check round X"`
- `run_in_background: true`
- `prompt`: 下記「Agent への指示テンプレート」に、以下を埋め込んで渡す：
  - `X`（ラウンド番号）
  - 対象 `.tex` ファイルの絶対パス一覧
  - `.bib` ファイルの絶対パス（検出できた場合）
  - 数値検証スクリプトの絶対パス（検出できた場合）
  - `<cwd>/.claude/proofread.md` の内容（プロジェクト固有の既知数値・追加ルール）
  - 出力先の絶対パス `<cwd>/doc/math_proof_check_X.txt`

### Step 5. ユーザーへの通知

Agent 起動後、即座に報告：

- バックグラウンドで第 X 回数学検証を開始したこと
- 出力先（絶対パス）
- 完了時に通知される旨

Agent の進捗を polling しない。

### Step 6. 連続ゼロ件への到達判断（任意）

ユーザーが「問題ゼロの回が N 回続くまで繰り返す」と指示した場合は、完了通知を受けて以下を判定：

- 直近 N 回のレポートがすべて Critical 0 / Major 0 / Minor 0 / Style 0 ならば終了
- 1 件でも検出があれば、ユーザーに修正プランを提示 → 修正後に本スキルを再起動
- 次回の Agent には「前回と異なる独立視点で検証せよ」と明示指示することで検証の多様性を確保（同じ切り口を繰り返さない）

## Agent への指示テンプレート

以下を Agent に渡す。`{{...}}` はスキル側で置換する。

```
あなたは数学論文の査読者・検証者です。日本語で作業してください。以下の .tex ファイルを数学的厳密性の観点から徹底検証し、結果を指定の出力ファイルに保存してください。

【対象】
{{target_tex_files_abs_paths}}

【.bib ファイル】（存在する場合のみチェック）
{{bib_file_abs_path_or_none}}

【数値検証スクリプト】（存在する場合のみ参照。独立再計算のクロスチェック材料）
{{verification_scripts_abs_paths_or_none}}

【出力先】
{{output_file_abs_path}}

【プロジェクト固有設定】
{{contents_of_cwd_dot_claude_proofread_md_or_none}}

【絶対の制約】
- 対象の .tex ファイルは絶対に直接編集しない。Edit/Write は出力ファイル（math_proof_check_X.txt）にのみ使用する。
- LaTeX のビルド・コミット・プッシュはしない。
- 時間がかかってもよいので徹底することを優先する。

【参照すべき情報】
- ~/.claude/CLAUDE.md の「Mathematical notation」「Statistical terminology」などのユーザー共通ルール
- <cwd>/CLAUDE.md のプロジェクト固有ルール
- <cwd>/.claude/proofread.md のプロジェクト設定（既知数値・追加ルール・優先事項）
- <cwd>/doc/math_proof_check_*.txt の直近ファイル（過去の指摘と重複を避けるため）。ただし過去ラウンドの結果に引きずられず、独立視点で再検証すること。
- <cwd>/doc/proof_reading_*.txt の直近ファイル（proofread 系との役割分担の把握）
- ユーザー auto memory（~/.claude/projects/<project-slug>/memory/ 配下）

【検証観点】

## A. 証明の論理的完結性
付録・本文の全命題・観察・系・補題・主張の証明を 1 つずつ追い、以下を検証：
- ステップの飛躍・未定義記号・符号の取り違え
- 陰関数定理の適用条件（ヤコビアン非退化）が明示されているか
- KKT 条件・ラグランジアンの符号（ラグランジュ乗数の非負性等）
- 仮定（asm1, asm2 等）が証明のどこで使われているか、必要十分性
- 単調尤度比・対数線形性・1 次確率的支配の正しい使用
- 境界ケース・退化ケースの扱い（σ→0, σ→∞, p_H→p_L, 仮定が等号で成立する場合など）

## B. 命題本体と証明結論の整合
- 命題の主張文と証明が導く結論が一字一句整合しているか
- 狭義不等号・広義不等号の区別
- 「少なくとも一方が狭義」「両方狭義」等の前提条件

## C. 定義と使用の整合
- 補助記号（$\Phi_H$, $\Phi_L$, $\phi_H$, $\hat h$, $\hat\gamma$, $\delta$, $R_H^{\mathrm{bd}}$ 等）の定義と使用が矛盾ないか
- 同一記号の二重定義（例: 無条件版の $M$ と条件付き版の $M$）がないか
- 装飾記号（$\ast$, $\star$, $\dagger$, $\bar{}$, $\hat{}$, $\check{}$, $\tilde{}$）の使い回しが混乱を生んでいないか

## D. 数値カリブレーションの独立再計算
- プロジェクト設定または既出のレポートに記載された数値例を Python/scipy または Julia で独立再計算
- 本文の丸め桁数と再計算値が整合するか
- 感応度分析表・ケース比較表の数値間の整合性（符号・大小関係）
- 独立な数値検証スクリプト（bin/ 以下）がある場合は実行して現状の本文命題と整合するか確認

## E. 相互参照の整合
- `\ref{}`, `\eqref{}`, `\cite{}` がすべて有効か
- 未定義参照・未使用ラベル・重複ラベル
- `\tag{\ref{eq:N}}` 等の二重参照が正しく機能するか
- ラベル命名規則の一貫性（eq:1, eq:2, ... の連続性、`eq:asm1` 等の名前付きラベルとの区別）

## F. 記法規則の遵守
`~/.claude/CLAUDE.md` の Mathematical notation セクションに従う：
- `\geqslant`, `\leqslant`（`\ge`, `\le`, `\geq`, `\leq` 禁止）
- `\log`（`\ln` 禁止）
- `\exp(x)`（`e^x` 禁止）
- `\Pr(`, `\E(`（角括弧禁止）
- `\text{subject to}`（`\text{s.t.}` 禁止）
- align 環境（equation 環境禁止）
- `\equiv`（`\overset{\mathrm{def}}{=}` 禁止）
- 略語的な複数文字記号は `\mathrm{}` で立体化（IC, FFP, HN, LN, HD, LD, PC, CE, bd, ch 等）
- インライン `$...$` 内の `\frac` 禁止
- 参加制約タグは `PC`（`IR` 禁止）

## G. Abstract・Introduction・Conclusion の方向性整合性
- 要旨・はじめに・おわりにで述べる定性的主張が命題と矛盾していないか
- 数式・文献引用の排除（要旨）、数式・統計量の排除（はじめに・おわりに）

## H. 実証節との整合性（実証仮説を含む論文の場合）
- 実証仮説が命題・系から正しく導出されているか
- モデル・パラメータと実証代理変数の対応が具体的か

【重大度分類】
- **Critical**: 論理破綻、証明が成立しない、命題が誤り
- **Major**: 主張の方向性の誤り、命題本体と証明結論の不一致、参照切れ
- **Minor**: 細部の記述ミス、記号の不一致、軽微な不正確さ
- **Style**: 記法規則違反・明瞭化推奨

【手順】
1. 対象 .tex をすべて全文 Read（長大なら offset/limit で複数回に分割）
2. .bib と数値検証スクリプトを Read
3. 直近の math_proof_check_*.txt と proof_reading_*.txt、CLAUDE.md、memory を Read
4. Grep で記法規則違反を網羅検出（F 項の全パターン）
5. ラベル・参照・引用の対応表を独立に作成し、未定義参照・重複・未使用を検出
6. 全証明を 1 つずつ追跡し、論理・符号・仮定の使用を検証
7. 数値例は /tmp/ に Python/Julia スクリプトを書いて独立再計算。本文の丸め桁数と一致を確認
8. 境界・退化ケースで命題の主張が破綻しないかを数値・代数的に検証
9. 独立な数値検証スクリプト（bin/ 以下）があれば実行
10. レポートを Write で出力ファイルに保存

【本ラウンドの独立視点（過去ラウンドとの差別化）】
過去の math_proof_check が既に実施済みの場合は、**別の切り口**で独立検証すること。たとえば：
- ラウンド N-1 が「代数検算中心」ならラウンド N は「境界ケース・退化ケース中心」
- ラウンド N-1 が「scipy で数値検算」ならラウンド N は「sympy で記号検算」「Julia で独立実装」
- ラウンド N-1 が「付録の証明追跡」ならラウンド N は「仮定の必要性・独立性追跡」
同じ検算を繰り返すだけでは検証の多様性が得られない。
```

## `.claude/proofread.md` の共通形式（proofread-paper と共有）

プロジェクト固有情報は `<cwd>/.claude/proofread.md` に書き、proofread-paper と math-proof-check 両方で使う：

```markdown
# プロジェクト校正設定

## 対象ファイル（相対パス）
- main.tex
- online_appendix.tex

## 既知数値（各ファイル内の一貫性チェック用）
- γ* ≈ 6.9748（5 桁表記）
- δ ≈ 0.3950（5 桁表記、論文定義）
- 例1: ∂γ*/∂σ = -1.28155、∂δ/∂(x_H-x_L) = 0.0020505

## ノーテーション方針
- 外生変数=小文字ラテン、戦略変数=小文字ギリシア、添字=大文字
- 補助関数・補助記号は「長い項が繰り返し出現する場合のみ」導入

## 数値検証スクリプト
- bin/verify_propositions.jl — 全命題を広範な外生パラメータで Julia 検証（最新 pass=73170/0）
- bin/calibration.py — 基本カリブレーションを scipy で数値生成

## 追加の優先指摘事項
...
```

スキル側はこのファイルを無加工で Agent プロンプトに埋め込む。

## 関連スキル

- `proofread-paper`: 文章表現・引用・体裁の校正。数学的厳密性は扱わない。
- **組み合わせ運用推奨**: 大改訂前は `math-proof-check` でゼロ件到達を確認 → `proofread-paper` で仕上げ、の順が効率的。証明の数学的問題と文章表現の問題は別レイヤーで、別個に繰り返すのが実績のあるワークフロー。
