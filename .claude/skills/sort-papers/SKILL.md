# 論文PDF振り分け・スキル

`~/papers/work/` にあるPDFを、ファイル名の最初の1文字（大文字に正規化）に対応する `~/papers/<A-Z>/` フォルダへ移動します。

## 手順

以下を**確認を求めず一気に**実行する。

### 1. workディレクトリ内のPDFを列挙

```
ls ~/papers/work/*.pdf
```

### 2. 各PDFの移動先を決定

- ファイル名の最初の文字を取得し、英字なら大文字に変換する。
- 移動先フォルダは `~/papers/<X>/`（X はその大文字）。
- 最初の文字が英字でないPDFは対象外として報告する（移動しない）。

### 3. 衝突チェック

各PDFについて、移動先に同名ファイルが既に存在するか確認する。存在する場合は **古いほうを残すために移動しない**。`mv -n`（no-clobber）を使うか、`test -e` で事前チェックして移動を行なう。

推奨は `mv -n` の利用：

```
mv -n source dest
```

ただし `-n` は黙って失敗するので、移動前後で source の存在を確認し、結果を判定する。あるいは事前チェック方式：

```
if [ ! -e "~/papers/B/file.pdf" ]; then
  mv "~/papers/work/file.pdf" "~/papers/B/"
fi
```

### 4. 一括移動

全ファイルの移動を1つのBashコマンドで実行する。`&&` ではなく `;` で繋ぐ（途中で1つスキップしても続行できるように）。

例:

```
cd ~/papers/work/ && \
  mv -n "Ball_Shivakumar_2006_JAR.pdf" ../B/ ; \
  mv -n "Demski_1974_TAR.pdf" ../D/ ; \
  ls
```

### 5. 報告

各ファイルについて以下のいずれかを簡潔に一覧報告する:

- **移動済み**: `file.pdf` → `<X>/`
- **スキップ（衝突）**: `file.pdf` は `<X>/file.pdf` が既存のため移動せず
- **対象外**: `file.pdf` は最初の文字が英字でないため移動せず

## 注意

- ファイル名の最初の文字のみで振り分ける（著者の last name の頭文字と一致するはずだが、特殊な場合でも文字どおり判断する）。
- 衝突時は **必ず古いほう（移動先側）を残す**。新しいほう（work側）は work に残ったままにする。
- 移動先フォルダが存在しない場合は作成しない。報告のみ行なう（papers/直下にA-Zフォルダがある前提）。
- アクセント記号付きの文字（É, Ñ 等）は対応するアルファベット（E, N）にマップせず、文字どおり大文字化を試みる。判断に迷う場合はユーザーに確認する。
